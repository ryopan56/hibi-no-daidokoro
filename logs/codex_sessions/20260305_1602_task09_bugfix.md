## 指示内容
- requirements に openai を追加してクリーンビルドで ImportError を避ける。

### 2) 実装概要（仕様→実装の対応）
- `select_for_update()` は必ず `transaction.atomic()` 内で実行。
- `used_count < 3` 判定と `used_count + 1` を同一 atomic ブロックで処理。
- `used_count >= 3` の場合は OpenAI を呼ばず `rate_limited` 応答＋`AiUsageLog` 記録。
- 枠確保後にトランザクションを抜け、OpenAI 呼び出し（ロック外）を実行。
- OpenAI失敗/タイムアウト/入力不正/SDK不在はすべて `fallback` でスキーマ準拠応答。

- `docker compose exec web python manage.py test`

### 4) 注意点
- 失敗時も枠は消費する設計（枠確保を先行）
- 日次集計は `AiDailyUsage`、監査ログは `AiUsageLog` で用途分離
