# Task09 セッションログ

## 指示内容
- `/ai/minimum` と `/ai/recommend` の POST API を実装。
- Structured Outputs（スキーマ準拠 JSON）で返却。
- JST 基準でユーザー単位 1 日 3 回制限（minimum/recommend 合算）。
- `ai_usage_logs` に毎回記録。
- 制限時・エラー時・タイムアウト時も必ずフォールバック JSON を返却。

## Codexの応答
- `ai` アプリを新規追加し、モデル・ビュー・URL・サービス・テストを実装。
- OpenAI 呼び出しを `ai/services/ai_client.py` に隔離し、JSON Schema 指定の Structured Outputs 呼び出しを実装。
- レート制限、フォールバック、ログ記録を共通ハンドラに統合。
- `TIME_ZONE` を `Asia/Tokyo` に調整し、`timezone.localdate()` 判定を採用。
- `check` / `test` を実行し成功を確認。

### 2) 実装概要（仕様→実装の対応）
- `/ai/minimum` `/ai/recommend` を `login_required + require_POST` で実装。
- JSON 入力を正規化し、必須キー付き JSON（status/mode/message/suggestions）を常時返却。
- `AiUsageLog` へ毎リクエスト1行を保存（ok/fallback/rate_limited）。
- `timezone.localdate()` で JST 日付を使い、1日3回制限を mode 合算で判定。
- 制限超過時は OpenAI を呼ばず `rate_limited` 定型メッセージと固定簡易案を返却。
- APIエラー/タイムアウト/予期せぬ例外/入力不正時は `fallback` 定型メッセージと固定簡易案を返却。
- OpenAI 呼び出しは `ai/services/ai_client.py` に分離し、`responses.create(... text.format.type=json_schema)` で Structured Outputs 指定。
