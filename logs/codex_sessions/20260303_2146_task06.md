# Task06 セッションログ

## 指示内容
- Task6 食材カテゴリ（中間テーブル）を追加し、ログ画面にチェックボックスCRUDを実装。
- SSOT準拠の enum（MEAT/FISH/EGG/VEGETABLE/BEAN/FROZEN, DB値0-5）。
- UniqueConstraint(meal_log, category)。
- 既存のログ方針に合わせたログ出力。
- マイグレーション/テスト実行。

## Codexの応答
- IngredientCategory を IntegerChoices で追加。
- MealLogIngredient モデル（category SmallIntegerField choices、UniqueConstraint、created_at）を追加。
- ログ画面でチェックボックス保存/更新/解除を実装。
- テストを追加し、手動確認手順を整理。
- ローカルで Django 実行不可のため手作り migration を作成し、Docker環境で検証。

## 実施変更
- `src/meallogs/enums.py`: IngredientCategory を IntegerChoices で追加。
- `src/meallogs/models.py`: MealLogIngredient モデル追加。
- `src/meallogs/forms.py`: ingredient_categories のチェックボックスフィールド追加。
- `src/meallogs/views.py`: カテゴリの保存/更新/削除とログ出力追加。
- `src/meallogs/templates/meallogs/log_detail.html`: チェックボックス表示を追加。
- `src/meallogs/tests.py`: ユニーク制約/保存更新/権限をテスト。
- `src/meallogs/migrations/0004_meallogingredient.py`: 中間テーブル作成のマイグレーション追加。

## 発生問題
- sandbox 環境に Django がなく `manage.py` コマンドが失敗。
- Docker デーモン未起動で `docker compose run` が失敗。

## 解決方法
- migration を手作りで作成し、ユーザー側 Docker 環境で `makemigrations/migrate/test` の結果が問題ないことを確認。
