# Task09 追加バグ修正（AiDailyUsage seed）セッションログ

## 指示内容
- AiDailyUsage 初回作成時に、既存の当日 AiUsageLog を使って used_count を seed する。
- seed 対象は `status in ("ok", "fallback")` のみ（`rate_limited` は除外）。
- seed → `used_count>=3` 判定 → `used_count+1`（枠確保）を同一 `transaction.atomic()` 内で実行。
- OpenAI 呼び出しはロック外のまま維持。

## 実施変更
- `src/ai/views.py`
  - `_lock_daily_usage` を `(daily_usage, created)` を返す形に変更。
  - `_reserve_daily_quota` 内で `created=True` のときのみ seed を実施。
    - `seed_count = AiUsageLog.objects.filter(user=user, jst_date=jst_date, status__in=["ok", "fallback"]).count()`
    - `daily_usage.used_count = min(seed_count, DAILY_LIMIT)`
    - `daily_usage.save(update_fields=["used_count", "updated_at"])`
  - 上記後に同一 atomic 内で上限判定と枠確保を実行。
- `src/ai/tests.py`
  - `test_seed_from_existing_usage_logs_when_daily_usage_missing` を追加。
  - 既存当日ログ2件(ok/fallback) + rate_limited1件の状態から、
    - 1回目は許可（ok/fallback）
    - 2回目は `rate_limited`
    を検証。
