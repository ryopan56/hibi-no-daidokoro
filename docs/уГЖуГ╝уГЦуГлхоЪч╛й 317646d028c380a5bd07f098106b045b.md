# テーブル定義

## テーブル定義

### 共通（タイムゾーン）

`created_at/updated_at` をUTC運用にする方針

Django側は原則として：

- `USE_TZ = True`
- `TIME_ZONE = "UTC"`（※週次通知判定はJST基準という要件はアプリロジック側で扱う）

## 3. photos

技術スタック側で「Djangoの `MEDIA_ROOT/MEDIA_URL`、ローカル保存、将来S3差し替え」を採用。なので `file_path(VARCHAR)` より、Django流に **`ImageField`（中身はパスを保持）** 。

（※「最大10枚」などの制約はDBではなくアプリ側でチェックする）

| 論理名 | Djangoフィールド | Null | 制約/備考 |
| --- | --- | --- | --- |
| id | `BigAutoField` | × | PK |
| meal_log | `ForeignKey(MealLog, on_delete=CASCADE, related_name="photos")` | × |  |
| image | `ImageField(upload_to="photos/%Y/%m/%d/")` | × | ファイル名はUUID運用（要件：UUID） |
| created_at | `DateTimeField(auto_now_add=True)` | × |  |

| 論理名 | Djangoフィールド | Null | 制約/備考 |
| --- | --- | --- | --- |

| created_at | `DateTimeField(auto_now_add=True)` | × |  |
| 制約 |  |  | `UniqueConstraint(fields=["meal_log","category"])` |
