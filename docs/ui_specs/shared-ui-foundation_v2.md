# shared-ui-foundation v2

## 概要

- Task12「共通 UI 基盤とレイアウトシェル整備」の Pencil フロー用 mock v2
- v1 の静かさは残しつつ、「四角い」「温かみがない」「ほっとしない」という指摘に対して、形・余白・色・面の扱いを見直した版
- artifact-slug: `shared-ui-foundation`
- version: `v2`

## v1 からの変更点

- 画面フレーム、カード、入力欄、通知、CTA に丸みを追加した
- 背景を白黒グレー中心から、ややベージュ寄りのやわらかい中間色へ寄せた
- カード間の呼吸を増やすため、余白と gap を少し広げた
- auth shell に導入用のやわらかい面を追加し、入口の緊張を下げた
- signed-in shell の歓迎メッセージを banner 感よりも「そっと置かれた面」に寄せた
- 下部ナビは完成形としては見せず、受け皿としての placeholder 感を残した
- flash / error の赤緑の強さを少し抑え、面と角でやわらかさを出した

## v2 内の再調整点

- 薄すぎた補助文字、help text、補助ラベルのコントラストを 1 段上げた
- signed-in shell では `primary action` の背景と CTA の強さを少し戻し、secondary 領域との差を明確にした
- auth shell 上部の導入カードは残しつつ、面を薄く、文量を短くして、ログインまでの流れを短くした

## 四角さをどう緩和したか

- 画面外枠に大きめの corner radius を入れた
- カード、通知、入力欄、CTA は 14〜18px 程度の丸みを持つ前提で整理した
- 下部ナビもピル型の受け皿にして、硬い箱感を減らした
- ただし装飾的な曲線にはせず、静かな整列感は維持している

## 温かみをどこで出したか

- 全体の地色を真っ白ではなく、少しだけベージュが混ざる背景にした
- surface も純白から少しだけやわらいだ白へ寄せた
- success / error / 補助面は、強い UI トーンではなく、台所の空気に近い淡い色へ落とした
- 影はほのかな外側 shadow のみで、硬いコントラストを避けた

## ほっとする感じをどこで作ったか

- auth shell の上部に「まだ急がなくていい」と感じられる薄い面を用意した
- 歓迎メッセージは注意喚起バナーではなく、小さな居場所のような面で扱った
- CTA の周囲に余白を取り、急かされる密度を避けた
- form error も「刺す赤」ではなく、やわらかい面の中で位置を明確にする方向にした

## フレーム構成

1. `Auth Shell`
2. `Signed-in Shell`
3. `Flash Message Pattern`
4. `Form Error Pattern`

## auth shell と signed-in shell の方向性

### auth shell

- 未ログイン時の入口
- 主目的は login / signup
- 上から静かに読ませる
- world view を最小限に含みつつ、フォームの緊張を下げる

### signed-in shell

- ログイン後の土台
- 主 CTA を最も強く見せる
- 二次導線は別面へ逃がす
- 共通ヘッダーと下部ナビの受け皿を持つ

## フラッシュメッセージの方針

- success / info / error は同一構造を維持する
- message は短く、断定しすぎず、責めない
- placement はヘッダー直下またはフォームカード上で固定する
- 色で殴らず、面と位置で理解させる
- 補助説明の文字色は薄くしすぎず、実際に読めることを優先する

## フォームエラーの方針

- non-field error はフォーム先頭にまとめる
- field error は対象フィールド直下
- help text と error text は同じ位置を使う
- border だけで強く責めず、背景面でも受け止める

## 下部ナビをどこまで確定したか

- v2 では「存在する受け皿」まで
- 項目数、最終ラベル、アイコン構成は確定しきらない
- active 1 / placeholder 2 の見せ方に留めて、共通 shell の議論を優先している

## 実装時の注意点

- Task12 は共通土台なので、個別画面固有の情報量をここで増やしすぎない
- 丸みや色味は CSS token 化し、画面ごとにばらつかせない
- auth と signed-in の差は layout shell で出し、各ページごとの独自判断は最小限にする
- flash / error の構造は共通 partial 化しやすい形を保つ
- 下部ナビは実装時も placeholder を超えて確定しすぎないよう注意する
- `login_id` / `display_name` 前提は維持する
- helper text と supplementary label は、デザイン優先で薄くしすぎない
- `primary action` と `secondary` の視覚差は、色・面・余白で保ちつつ、派手にはしすぎない
- auth shell 上部の導入要素は、ログイン導線を止めるほど強くしない

## 思想 / ペルソナとの接続

- 「義務からの解放」「判断疲れを軽くする」を、急かさない余白とやわらかい面で表現する
- ペルソナが夕方に少し立ち止まる感覚を想定し、UI もすぐ決断を迫らない
- 「ここに来ただけで十分」という感触を、文言だけでなくレイアウトの圧の低さでも作る

## 人間承認時に見たいポイント

- v1 よりやわらかくなったが、装飾過多にはなっていないか
- auth shell と signed-in shell の役割差が自然に見えるか
- flash / error が still calm で、弱すぎず強すぎないか
- 下部ナビが確定しすぎていないか
- 「ほっとする」方向が、文言ではなく見た目でも出ているか
- 補助文字がちゃんと読めるか
- primary と secondary の差が、やわらかさを保ったまま見分けられるか
