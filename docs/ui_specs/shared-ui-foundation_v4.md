# shared-ui-foundation v4

## 概要

- Task12「共通 UI 基盤とレイアウトシェル整備」の Pencil フロー用 mock v4
- v3 の静かさ、やわらかさ、ほっとする感じは維持しつつ、固定背景の導入と Startup Copy Layer の軽量化を行った版
- artifact-slug: `shared-ui-foundation`
- version: `v4`

## v3 からの変更点

- `Startup Copy Layer` をカード感の強い見え方から、白背景に近い面の中へ文字がふわっと現れる前提へ寄せた
- auth / signed-in に残っていた肯定メッセージ box を削る、または極小化し、肯定メッセージの主役を Startup 側へ集約した
- 全フレームで使える固定背景を導入し、地味さを少し和らげつつも、UI 要素より前に出ない静かな気配を追加した
- flash / error はそのまま役割を分けつつ、固定背景の上でも読みやすいよう「面で受ける」見え方をそろえた

## フレーム構成

1. `Startup Copy Layer`
2. `Auth Shell`
3. `Signed-in Shell`
4. `Flash Message Pattern`
5. `Form Error Pattern`

## Startup Copy Layer をどう整理したか

- v3 よりも「前段レイヤー」であることを優先し、重い導入カードにはしない
- 白背景に近い静かな面の中へ短いコピーが現れて消える前提を示す
- 起動時に登録済みテキストからランダム表示されること、その後に auth または signed-in shell が出ることだけを伝える
- 秒数やアニメーションの細部は、この mock の段階では固定しない

## auth / signed-in から何を削ったか、弱めたか

- auth の肯定メッセージ box は独立した主役面としては置かず、薄い導入テキストに整理した
- signed-in の歓迎メッセージ box も常設しない
- signed-in では `primary action` を最初に見せ、二次導線と下部ナビは受け皿に留める
- 肯定感は Startup で先に渡し、その後の shell では行動と案内を中心にする

## 固定背景の意図

- v3 の「少し地味」という印象を和らげるため、全画面共通で使える静かな背景を入れた
- 表現は柄やイラストではなく、ごく薄いトーン差と淡いグローだけに留める
- 背景単体で意味を持たせず、気配として奥に置く

## 固定背景が共通基盤にどう効くか

- auth / signed-in / flash / error のどのフレームでも破綻しにくい
- ベーステンプレート実装時に、ページ固有要素を足しすぎなくても画面の温度を保ちやすい
- メッセージやフォーム面を「白に近いサーフェス」で受ける前提にすると、背景と UI の階層を安定させやすい
- 後続画面でも、共通 shell の印象を保ちながら内容差分を載せやすい

## flash / error との関係

- 肯定メッセージは Startup 側へ集約し、常設の flash と混ぜない
- `success flash` は操作結果の通知
- `info / guidance` は補助案内
- `form error` は修正ポイントの提示
- いずれも背景の上で直接見せるのではなく、共通の面の中に置いて読みやすさを保つ

## 下部ナビをどこまで確定したか

- v4 でも「受け皿」まで
- active 1 と placeholder を置く程度に留める
- 項目数、最終文言、遷移設計は確定しない

## 実装時の注意点

- 固定背景は共通 base / shell 側で管理し、個別画面ごとに強弱をばらつかせない
- 背景は CSS 実装時もコントラストを抑え、コンテンツより前に出さない
- Startup Copy Layer は一時表示レイヤーとして扱い、shell 内の常設メッセージで代替しない
- auth / signed-in では主 CTA と補助導線の優先順位を崩さない
- Task12 の段階なので、個別画面の完成度を上げに行かず、共通受け皿の見え方に集中する
- `login_id` / `display_name` 前提は維持する

## 承認時に見たいポイント

- 固定背景が気配として効いているか
- Startup Copy Layer が軽くなり、前段レイヤーとして読めるか
- auth / signed-in から肯定 box を減らしても、冷たくなっていないか
- flash / error が背景の上でも見分けやすいか
- 下部ナビが引き続き受け皿レベルに留まっているか
